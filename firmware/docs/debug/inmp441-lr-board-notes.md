# INMP441 LR 板级异常排查

日期：`2026-04-13`

适用范围：

- 当前用户手上的异常板
- 不代表另一块开发板或所有 `bbclaw` / `breadboard` 板型都会复现

## 现象

- 语音解锁录音时长有时只有 `2s ~ 3s`
- 即使按住 PTT 说得更久，送去验证的 PCM 仍可能被截到 `4s`
- 同一套固件在另一块开发板上表现正常
- 某些样本出现左右声道自动锁定不稳定，录音质量明显波动

## 本次新增观察

- `INMP441` 模块的 `L/R` 引脚与异常高度相关
- 这块异常板上：
  - `L/R -> GND` 时表现异常
  - `L/R` 悬空后表现反而恢复/明显变好

## 当前判断

- 这更像是板级硬件差异、接线、模块个体状态或 `L/R` 管脚电平质量问题
- 现阶段不把“`L/R` 必须接 `GND`”当作通用结论
- 对这块异常板，`L/R` 悬空是一个有效的临时绕过方案

## 与固件现状的关系

- 固件当前启用了 `BBCLAW_AUDIO_RX_AUTO_CHANNEL_LOCK=1`
- 因此 `INMP441` 的左右声道选择会受实际输入能量影响
- 如果 `L/R` 电平异常、悬浮状态不稳或模块输出异常，自动锁定可能选到错误一侧

## 后续建议

1. 继续保留这块异常板与正常开发板做 A/B 对照
2. 对异常板优先记录：
   - `L/R -> GND`
   - `L/R -> 3V3`
   - `L/R` 悬空
3. 结合串口日志重点看：
   - `pick_right=0/1`
   - `rms`
   - `truncated`
4. 在未确认根因前，不把这条经验直接写进通用接线结论
