# BBClaw Roadmap

## 硬件形态（软件能力）

**体验灵感**来自传统对讲机（按住说话、实时语音）与 BB 机/寻呼机（推送、轻量摘要）— 同一套固件在一条产品里承载这些能力，**不是**「对讲 / 通知」二选一的两种模式切换。

| 能力面 | 描述 | 状态 |
|------|------|------|
| **实时语音（PTT 链路）** | 按下按键采集上传，经 Adapter ASR 等与 Gateway 交互 | ✅ 已实现 |
| **通知与摘要** | 异步推送、轻量展示与触觉提示等 | ✅ 已实现 |
| **体验整合** | 语音与通知在同一设备上自然呈现，持续打磨 | 🔄 持续优化 |

## 外设与 v1.0 硬件路线

当前实现与 **v1.0 目标形态**一致：**开发板极客方案 + 3D 打印外壳**，以可复现、可迭代为先。

| 类别 | 典型选型 / 说明 |
|------|------------------|
| **MCU** | ESP32-S3 |
| **音频** | ES8311 CODEC，麦克风 / 扬声器与功放链路 |
| **显示** | 1.47″ ST7789，面板 172×320，横屏 320×172，LVGL |
| **交互** | PTT 按键、振动马达（触觉反馈） |
| **连接** | Wi-Fi |

**v1.0 不主打**：为追求体积而单独开模、重画小型化定制 PCB；先把软件、协议与体验跑稳，用现成模块 + 打印外壳完成「可玩的 1.0」。

## 中长期：外壳 IP

产品化下一阶段的重点是 **外壳与品牌体验（外壳 IP）**，与本仓一致的方向是：

- **开源固件**（本仓）
- **开源外壳 / 结构**（社区可打印、可改进，与固件配套发布）

运行拓扑与扩展方案见 `docs/architecture.md`。

## 连接方案（当前已验证）

### WiFi 局域网（✅）

```
BBClaw 设备 ──WiFi──► 同一内网 ◄──WiFi── 运行面 (Adapter)
     │                                    │
     │  典型配置:                          │
     │  - WiFi SSID/Password              │
     │  - Adapter IP 地址                  │
     ▼                                    ▼
   音频/控制  ←─────── LAN ────────►  转发 → OpenClaw Gateway
```

**优点**：延迟低、带宽足。  
**代价**：需配置 WiFi 与运行面可达地址。
